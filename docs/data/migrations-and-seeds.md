# Данные: миграции и сиды

Документ описывает, как управляются миграции и наполнение базы в текущем состоянии проекта.

## Текущее состояние

- В репозитории **нет файлов миграций**.
- Таблицы создаются на лету в процессе CLI импорта адресов.
- `TypeORM` настроен на поиск миграций в `dist/infrastructure/database/migrations`, но каталог пуст.

## Как инициализировать базу

Единственный поддерживаемый способ — запуск импорта:

```bash
npm run import:addresses:dev
```

Импорт сам создаёт таблицы, индексы и расширения. Подробнее см. в [Данные: импорт адресов](addresses-import.md).

## Если понадобятся миграции

1) Определите стратегию: что должно делать импортом, а что — миграциями.
2) Добавьте миграции и скрипты их запуска в `package.json`.
3) Зафиксируйте процесс в документации.

Важно: пока миграций нет, любые изменения схемы фиксируются в коде импорта.
