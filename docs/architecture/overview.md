# Архитектура: обзор

Документ описывает общую структуру сервиса, ключевые слои и принципы взаимодействия модулей.

## Ключевые компоненты

- **HTTP API**: основной NestJS процесс с контроллерами `addresses` и `health`.
- **CLI импорт**: отдельный входной процесс для загрузки адресов в PostgreSQL.
- **Инфраструктура**: общий логгер и подключение к базе через TypeORM.
- **Конфигурация**: централизована через `ConfigModule` и `zod` схему.

## Границы ответственности

- Модули в `src/modules/*` содержат бизнес-логику (контроллеры/сервисы/репозитории).
- Инфраструктурные модули находятся в `src/infrastructure/*`.
- CLI команды в `src/cli/*` не поднимают HTTP-сервер и используют `createApplicationContext`.
