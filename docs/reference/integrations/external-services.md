# Интеграции: внешние сервисы
Owner: Backend Team
Last reviewed: 2026-02-04

## Цель
Зафиксировать внешние зависимости, с которыми взаимодействует сервис.

## Контекст
В коде используются только HTTP загрузки адресного архива и SMTP для отправки OTP.

## Шаги
1. Проверьте адрес источника и доступность сети.
2. Настройте SMTP при необходимости.

## Источник данных адресов
CLI импорт загружает архив с данными адресов по HTTP(S) из сервиса бизнес-адресов:
- Базовый URL: `https://business.juso.go.kr/api/jst/download`.
- Параметры запроса формируются на основе `ADDRESS_DATA_MONTH`.
- Архив содержит файлы `road_code_total*.txt` и `build_*.txt`, которые обрабатываются импортом.

Подробнее — в [Импорт адресов](../../how-to/addresses-import.md).

## SMTP для OTP
- SMTP используется только если заданы `MAILER_HOST`, `MAILER_PORT`, `MAILER_FROM`.
- В production отсутствие SMTP приводит к ошибке `/auth/email/start`.

## Другие внешние сервисы
Прочих внешних интеграций в коде нет.

## Ограничения
Документ не описывает инфраструктурные интеграции (балансировщики, мониторинг и т.п.).
