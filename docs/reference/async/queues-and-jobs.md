# Асинхронные задачи
Owner: Backend Team
Last reviewed: 2026-02-04

## Цель
Зафиксировать состояние фоновых задач, очередей и cron-операций.

## Контекст
В коде нет очередей и cron-джоб. Единственный асинхронный процесс — CLI импорт адресов.

## Шаги
1. Используйте раздел для подтверждения отсутствия фоновых задач.
2. При появлении очередей/cron обновляйте документ.

## Текущее состояние
- Очереди (Bull/BullMQ), cron-джобы и event emitter не используются.
- Единственный асинхронный процесс — CLI импорт адресов.

## CLI импорт адресов
Импорт запускается командой `npm run import:addresses:dev` или `npm run import:addresses`.
- Не поднимает HTTP сервер.
- Использует advisory lock, чтобы не запускаться параллельно.
- Записывает прогресс в таблицу `address_import_state`.

Подробнее см. в [Импорт адресов](../../how-to/addresses-import.md).

## Если потребуются фоновые задачи
Рекомендуемые варианты:
- Cron/worker процесс (отдельный сервис).
- Очередь задач (Bull/BullMQ) с Redis.
- Планировщик во внешней инфраструктуре (например, Kubernetes CronJob).

## Ограничения
Раздел не описывает планировщики инфраструктуры — только код сервиса.
