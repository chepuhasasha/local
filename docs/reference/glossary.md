# Глоссарий
Owner: Backend Team
Last reviewed: 2026-02-04

## Цель
Зафиксировать основные термины проекта в одном месте.

## Контекст
Используйте глоссарий, если встречается термин из документации или кода и нужна точная трактовка.

## Шаги
1. Найдите термин в списке.
2. Если термина нет — добавьте его вместе с источником и ссылками.

## Термины
- Access-токен — JWT с `sub` (userId) и `sid` (sessionId), используемый в `Authorization: Bearer`.
- Refresh-токен — случайная строка, хранимая в базе только как SHA-256 хэш.
- OTP — одноразовый код для входа по email.
- Identity — запись `auth_identity`, связывающая пользователя и провайдера входа.
- Session — запись `auth_session`, отражающая refresh-сессию.
- Слепок адресов — набор таблиц `addresses`, `addresses_next`, `addresses_prev`.
- Shadow table — `addresses_next`, куда пишется новый импорт до переключения.
- `pg_trgm` — расширение PostgreSQL для триграммного поиска.
- Throttling — ограничение частоты запросов через `@nestjs/throttler`.

## Ограничения
Глоссарий не заменяет спецификацию API или схемы БД.
